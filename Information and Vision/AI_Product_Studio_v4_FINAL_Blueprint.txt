# 1. EXECUTIVE SUMMARY

AI Product Studio v4.0 is a private agency automation platform that transforms client ideas into revenue-generating digital products. Built for a single owner-operator, it handles end-to-end workflow: lead capture → business analysis → development → delivery — with mandatory human approval at every critical step.

Core Philosophy: "AI Suggests, Human Decides"

Key Differentiators:
- Rs. 0 cost for first 2-3 months (free-tier LLMs)
- 20 specialized workers across marketing, sales, research, planning, development, delivery
- Multi-LLM fallback (Gemini, Groq, Cerebras, OpenRouter, Ollama)
- MCP server integrations (Figma, Canva, Supabase, etc.)
- WhatsApp alerts for critical decisions
- Desktop app (Tauri) for local-first operation (Zero Server Cost)
- Private cloud sync for cross-device access

# 7. DATA ACCESS & IPC SPECIFICATIONS

All data interactions occur natively inside the React frontend via SQLite SQL commands executed through the Tauri IPC plugin (`@tauri-apps/plugin-sql`).

Instead of REST API endpoints, the application uses:
1. `getDb().execute(query, params)` for INSERT, UPDATE, DELETE.
2. `getDb().select(query, params)` for GET requests.
3. Local state (Zustand / React Context) for real-time reactivity.
4. Settings (`getSetting`, `saveSetting`) for API Keys.

Tauri IPC Commands (`invoke`):
- `invoke("gemini_proxy")`: Secure local proxy for LLM calls bypassing CORS.
- `invoke("llm_proxy")`: Secure local proxy for other LLM providers.
- Custom Rust functions mapped to frontend commands.

# 8. WORKER ARCHITECTURE (Cortex JS)

## 8.1 Worker Configuration

Workers live in `src/engine/workers/`. They are executed by the local `CronService` or `Cortex`.

| Worker | Queue | Approval | Severity | Priority |
|--------|-------|----------|----------|----------|
| self_promo | marketing | Yes | standard | 5 |
| lead_gen | sales | Yes | standard | 5 |
| business_analyst | research | Yes | standard | 5 |
| blueprint_maker | planning | Yes | standard | 5 |
| documentor | planning | Yes | standard | 5 |
| developer | development | Yes | standard | 5 |
| website_builder | development | Yes | standard | 5 |
| packager | delivery | Yes | CRITICAL | 10 |
... (And 15 more workers mapped to local JS classes).

## 8.2 Base Worker Structure (JavaScript)

```javascript
import { Cortex } from "../cortex.js";
import { getDb } from "../../data/db.js";

export default class BaseWorker {
  constructor(name, queue, requiresApproval = false, approvalSeverity = 'standard') {
    this.name = name;
    this.queue = queue;
    this.requiresApproval = requiresApproval;
    this.approvalSeverity = approvalSeverity;
    this.cortex = new Cortex();
  }

  async run(projectId, payload) {
    try {
      this.logStart(projectId, payload);
      const result = await this.execute(projectId, payload);

      if (this.requiresApproval) {
        await this.requestApproval(projectId, result);
      }
      this.logComplete(projectId, result);
      return result;
    } catch (err) {
      this.logError(projectId, err);
      throw err;
    }
  }

  async execute(projectId, payload) {
    throw new Error("execute() must be implemented by subclass");
  }
}
```

# 10. BUILD & DEPLOYMENT GUIDE

Since this is a Local-First Desktop App, there is no cloud server infrastructure to deploy (Rs. 0 hosting cost).

## Development
1. Install dependencies: `npm install`
2. Run development server (Browser): `npm run dev`
3. Run desktop shell (Tauri): `npm run tauri dev`

## Production Build
To create native executables (.exe, .dmg, .AppImage):
1. `npm run tauri build`
2. Distribute the generated binaries to the Owner.

## Data Archival (Cloud Sync)
- SQLite database holds active data locally.
- Data > 6 months is archived to the private cloud to keep the SQLite database fast.
